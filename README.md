# Audio Dataset Creation Tool

Это десктопное приложение на Python, предназначенное для быстрой и удобной записи аудио датасетов (например, для машинного обучения, распознавания речи или медицинских исследований аудио).

## Ключевые возможности
1. **Запись и метаданные**: Удобный графический интерфейс для заполнения информации об испытуемом (Имя, Состояние здоровья, ID).
2. **Точная синхронизация**: Приложение умеет генерировать высокочастотный звуковой сигнал (маркер), который синхронизирует начало отсчета времени как в аудиофайле, так и в файле логов (`.json`).
3. **Разметка файлов**:
   - Приложение генерирует .json файлы с метками времени. Внешний обработчик будет использовать их или детектировать ВЧ-сигнал.
4. **Горячие клавиши**: Управление записью и маркерами не отрывая рук от клавиатуры.
5. **Внешняя обработка (`standalone_trimmer.py`)**: В комплекте идет отдельный скрипт для пакетной обработки сохраненных сырых файлов (обрезка по маркеру, конвертация).

## Структура файлов проекта
- `main.py` — главный файл, содержащий графический интерфейс (Tkinter) и общую логику связи компонентов.
- `audio_engine.py` — модуль для асинхронной записи звука с микрофона и воспроизведения маркеров.
- `post_processor.py` — математический модуль поиска сигнала через фильтры Баттерворта и изменения формата файла (scipy, numpy).
- `run.bat` — быстрый скрипт для запуска программы в Windows одним кликом.
- `mic_calibration.py` — скрипт для проверки, способен ли ваш микрофон улавливать высокочастотный звук.
- `standalone_trimmer.py` — скрипт для обработки папок со сторонними аудиофайлами.
- `recordings/` — папка, куда по умолчанию сохраняются все сырые записи, обрезанные файлы (`_processed.wav`) и файлы разметок (`.json`).

## Формат сохранения файлов
Сырые аудиофайлы и JSON-файлы с таймстампами сохраняются в строгом формате:
`case_YYYY-MM-DD_MIC_<Название_Микрофона>_<Name>_Health_<ID>.wav`

Например: `case_2026-02-27_MIC_Realtek(R)_Ivan_Health_12345.wav`.
Это позволяет сразу понимать, кто, когда и на каком оборудовании был записан.

## Инструкция по использованию
1. Запустите приложение с помощью `run.bat`.
2. (Опционально) Заполните поля `Name` и `ID` в верхней части программы.
3. (Опционально) Нажмите **Load Text File** чтобы загрузить текст для чтения перед глазами.
4. Нажмите **Start Recording** (или клавишу `R`). Запись начнется, но таймер меток еще не пошел.
5. Нажмите **Play HF Tone** (или клавишу `S`). Прозвучит маркер, **с этого момента** начинается отсчет времени для JSON-меток.
6. В процессе записи нажимайте **Mark Timestamp** (или клавишу `Пробел`) для фиксации интересующих вас моментов времени.
7. Нажмите **Stop Recording** (или клавишу `Enter`).
8. Программа сохранит сырой аудиофайл и `.json` с метками. (Обрезка внутри приложения не производится!)

### Как обрезать записи?
Когда вы закончите создавать записи:
1. Запустите скрипт `.\venv\Scripts\python.exe standalone_trimmer.py` в терминале (или сделайте для него bat-файл).
2. Скрипт попросит указать путь к файлу или папке. Можете просто указать папку `recordings`.
3. Скрипт автоматически найдет маркеры во всех файлах, обрежет их, переведет в моно 16000Гц и сохранит с приставкой `_processed.wav`.

## Требования
Все зависимости (numpy, scipy, sounddevice, soundfile) устанавливаются автоматически при инициализации виртуального окружения `venv`. Для работы требуется Python 3.9+.
